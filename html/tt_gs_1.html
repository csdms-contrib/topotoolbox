Load a DEM and display it

Digital elevation models (DEMs) come in various formats. Most commonly, DEMs are regularly gridded and often are GEOTIFF files. TopoToolbox accepts *tif-files and ESRI ASCII grids as input to derive a variable of class GRIDobj. Here we use the example DEM that ships with the Toolbox. The DEM covers parts of the San Gabriel Mountains in California USA. It has a pixel spacing (spatial resolution) of 30 m and is projected to the UTM coordinate system.
Do you have your own data? Simply change the file address in following code to point to your DEM. This may require absolute file paths (e.g., 'C:/myfiles/.../DEM.tif').
DEM = GRIDobj('srtm_bigtujunga30m_utm11.tif');
You should have a new variable in your workspace called DEM. The variable is an instance of the TopoToolbox class GRIDobj and contains the elevation values as well as information about the projection.
DEM

Basic values relating to the DEM can be accessed with the function info.
info(DEM)

In case you have used your own file, you might get some warning about the projection. Did you supply a DEM with geographic coordinates? Don't you have the mapping toolbox?
Now let's plot the DEM. TopoToolbox has various functions to do this. Some of these functions are simply overloaded methods. This means, they are functions that are part of MATLAB but that can also be called with a GRIDobj as input.
imagesc(DEM)
h = colorbar;
h.Label.String = 'Elevation [m]';
A better impression of the topography can usually be gained using hillshading. The function imageschs (image with scaled colors and hillshading) plots a hillshade.
imageschs(DEM,[],'colorbarylabel','Elevation [m]','colormap','landcolor')
The function surf provides a 3D view of the DEM.
